浏览器缓存
2020-06-02
字数总计: 1.5k|阅读时长: 5 分钟

前言

浏览器缓存是浏览器的一个特性，而不是一个 bug, 它的存在可以方便我们在下次打开相同界面时提高网站的加在速度，让我们得到更舒适的体验~

缓存

静态网站浏览器缓存问题

大家都知道浏览器会对静态文件进行缓存，但是我们不妨思考一下这几个问题：

  • 静态文件为什么会被缓存？
  • 浏览器缓存是如何生效的？
  • 如何合理利用浏览器缓存？

浏览器强制缓存跟协商缓存

  1. HTTP 报文

    • HTTP 报文是浏览器和服务器通信时发送及响应的数据块
      • 浏览器 -> 服务器 （请求报文）
      • 服务器 -> 浏览器 （响应报文）
    • 报文主要分为两个部分
      • 包含属性的首部（header）——–附加信息(cookie, 缓存等)与缓存相关的信息都保存在 header 中
      • 包含数据的主体部分(body)——–HTTP 真正想传输的部分

  2. 强制缓存以及协商缓存

  • 浏览器在加载资源时，根据请求头的expires和cache-control判断是否命中强缓存，是则直接从缓存读取资源，不会发请求到服务器。
  • 如果没有命中强缓存，浏览器一定会发送一个请求到服务器，通过last-modified和etag验证资源是否命中协商缓存，如果命中，服务器会将这个请求返回，但是不会返回这个资源的数据，依然是从缓存中读取资源。
  • 如果前面两者都没有命中，直接从服务器加载资源。

强缓存与协商缓存的区别

  • 强缓存： 未将请求发送到服务器
  • 协商缓存： 将请求发送到服务器
