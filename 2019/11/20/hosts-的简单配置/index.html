hosts 的简单配置
2019-11-20
字数总计: 1.1k|阅读时长: 3 分钟

hosts 文件位置

C:\Windows\System32\drivers\etc里面的 hosts 文件

hosts 文件使用简介

1）通过记事本 打开 host 文件
2）host文件的作用是，保存 IP 地址和 Host name (主机名)的映射关系（所以，host 是一个映射 IP 地址和 Host name （主机名）的规定）
3）host文件内容编写规范： 一行只能写一个映射关系； IP 地址写在前，Host name （主机名）写在后；注释用#

为什么要配置 hosts

预备知识：（DNS（Domain Name System））

DNS(Domain Name System) 是域名系统。它是由解析器和服务器组成的。域名服务器是指保存有该网络中所有的主机域名和对应的 IP 地址，并具有将域名转化为 IP 地址功能的服务器。
NOW 了解一下 Hosts 在 Windows 下的工作过程

在网络上访问一个网站，首先要通过 DNS (Domain Name System) 服务器吧网络域名(www.XXX.com) 解析成 XXX.XXX.XXX.XXX的 IP 地址后，我们的计算机才能够访问。要是对于每一个域名请求，我们都要等待 DNS(Domain Name System)服务器 解析后返回 IP 信息，这样访问网络的效率机会降低。而 Hosts 文件存在的意义就是提高解析效率。根据 Windows 系统规定，在进行 DNS 请求之前， Windows系统会先检查自己的 Hosts 文件中是否有这个地址的映射关系， 如果有则先调用这个IP的映射关系， 如果没有，再向已知的 DNS 域名服务器提出域名解析。简言之，Hosts 的请求级别高于 DNS。

Hosts 文件的具体作用

1）加快域名解析
根据预备知识，我们可以知道， Hosts 文件的优先级要高于 DNS （Domaim Name System） 系统，所以，对于我们经常访问的网站，我们可以通过在 Hosts 文件中配置域名和 IP 的映射关系，使我们在输入域名时，可以很快解析出 IP 地址，而不用请求网络上的 DNS
（Domain Name System）服务器。
2）方便局域网用户
在很多单位的居于网中，会有服务器提供给用户使用，但由于局域网中一般很少架设 DNS （Domain Name System）服务器， 访问这些服务器要输入难记的 IP 地址，对于很多人来说比较麻烦。现在，可以通过给这些 IP 起一些容易记住的名字，然后在 Hosts 文件中建立 IP 映射，以后可以方便输入。
3）屏蔽网站
现在有一些网站不符合自己的需求，或者是网站本身存在着安全性的问题。对于这些网站，我们可以通过 Hosts 文件，把这些网站的域名映射到错误的 IP 或自己计算机的 IP, 从而达到屏蔽的作用


4）顺利连接系统
对于一些服务器要连接一些数据库服务器，在访问时如果直接输入 IP 地址是不能访问的，只能输入服务器名才能访问。这时，我们配置好 Hosts文件， 再输入服务器名字就可以顺利访问了
特别注意：Hosts 文件配置的映射是静态的，如果网络上的计算机更改了，请及时更新 IP 地址，否则将不能访问
